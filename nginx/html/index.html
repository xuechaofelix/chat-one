🚀 New API 用户使用指南

欢迎使用我们的 AI 接口集成平台！通过本平台，您可以使用一个统一的账号和格式，调用包括 OpenAI (GPT)、Anthropic (Claude)、Google (Gemini) 等在内的多种主流 AI 模型。

📌 核心使用流程

  1. 登录系统
  2. 使用兑换码充值余额
  3. 创建并获取 API Key (令牌)
  4. 查看额度与消耗记录
  5. 常用第三方 AI 软件接入指南

1. 登录系统

  1. 打开平台访问地址：https://chatone.site/
  2. 点击右上角的 登录 或 注册。
  3. 登录成功后，您将进入个人控制台首页。

2. 使用兑换码充值余额

在正式创建令牌调用接口前，您需要确保账户内有足够的可用额度。如果您购买或获取了充值兑换码，请按照以下步骤充值：

💳 充值步骤：

  1. 登录进入控制台，点击左侧导航栏的 「充值」 (Top-up)。
  2. 在 「兑换码充值」 输入框中，粘贴您获取到的兑换码（通常是一串长字符串）。
  3. 点击 「兑换」 按钮。
  4. 系统验证成功后，相应的额度将实时注入您的账户，您可以在页面右上角查看到最新的账户余额。

3. 创建并获取 API Key (令牌)

在使用第三方 AI 软件（如 Next Chat、Chatbox）时，您需要创建专属的密钥（Token）。

🛠️ 操作步骤：

  1. 点击左侧导航栏的 「令牌」 (Tokens)。
  2. 点击 「添加新令牌」 按钮。
  3. 设置令牌信息：
    • 名称：自定义（如：用于手机端、测试使用）。
    • 过期时间：默认永不过期，也可自定义。
    • 额度：可以设为“设为无限”（代表直接消耗您的账户总余额）或自定义具体额度（防止软件跑满误消耗）。
  4. 点击 「提交」 保存。
  5. 在令牌列表中，找到刚创建的令牌，点击 「复制」 或点击 「聊天」 直接跳转使用。

⚠️ 安全提示：请妥善保管您的 sk- 开头的密钥，切勿泄露给他人。

4. 查看额度与消耗记录

平台使用虚拟额度进行计费（通常 $1 对应一定的代币/Token 数量，根据模型定价决定）。

  • 实时额度：在系统右上方或 「首页」 可以看到您的当前剩余额度。
  • 消耗明细：点击左侧 「日志」 (Logs)，可以查看每一次调用的时间、使用的模型、消耗的额度以及是否调用成功。如果对消耗明细有异议，可以通过客服邮箱提交工单，申请官方账单比对。
  • 模型定价：点击系统上方的 「模型广场」，可以查询每个模型的定价。

5. 常用第三方 AI 软件接入指南

本平台完美兼容 OpenAI 官方接口格式。在任何支持自定义 API 的软件中，您只需要修改以下 两个核心参数：

参数名称 统一配置内容
API Base URL (接口地址) https://chatone.site/v1
API Key (密钥/令牌) sk-xxxxxxxxxxxxxxxxxx（您在步骤 3 复制的令牌）

📱 常见客户端配置示例：

① Next Chat (ChatGPT Next Web)

  1. 打开 Next Chat 的 设置 页面。
  2. 找到 模型服务商，选择 OpenAI。
  3. 打开 自定义接口 开关：
    • 接口地址 (Base URL)：输入 https://chatone.site/v1
    • API Key：粘贴您的 sk- 令牌。
  4. 在聊天界面选择您有权限使用的模型（如 gpt-4o, claude-3-5-sonnet）即可开始对话。

② Chatbox

  1. 打开 Chatbox 的 设置 -> 模型。
  2. 模型提供方选择 OpenAI API 或 自定义 (OpenAI 兼容)。
  3. 填入参数：
    • API 密钥：您的 sk- 令牌。
    • API 域名 (Base URL)：https://chatone.site/v1
  4. 选择需要的模型，点击保存。

❓ 常见问题与报错

  • ❌ 提示 401 Unauthorized / 全局额度不足：您的账户余额用尽，或者该令牌已被禁用/过期。请参考 步骤 2 使用兑换码充值。
  • ❌ 提示 兑换码无效/已被使用：请检查兑换码复制是否完整，或确认该兑换码是否已经兑换过。如仍有疑问，请联系您的发卡商或管理员。
  • ❌ 提示 403 Forbidden / 渠道不可用：当前选用的 AI 模型后台通道暂时出现故障，请尝试更换其他模型（例如从 gpt-4o 换成 gpt-4-turbo）或联系管理员。
  • ❌ 提示 400 Bad Request / 无法解析：请检查软件中的 Base URL 是否正确加上了 /v1 尾缀。
